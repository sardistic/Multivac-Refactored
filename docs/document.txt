The bot maintains conversation context by storing the conversation history in a SQLite database. Each conversation is identified by a unique conversation_id, which is a combination of the guild ID and the channel ID. This allows the bot to keep track of the conversation history for each channel in each guild.

When a user sends a message, the bot appends the user's message to the conversation history and then generates a response from the GPT-4 model based on this history. The bot then appends the generated response to the conversation history before sending the response back to the user in the Discord channel. This allows the bot to maintain a coherent context for the conversation, as the GPT-4 model generates responses based on the entire conversation history.

The bot does not share context between different users. Each user has their own conversation history, which is stored in a separate table in the SQLite database. The table name is user_{user_id}_history, where {user_id} is the ID of the user. This allows the bot to maintain separate conversation contexts for each user.

However, please note that the conversation context is shared between all channels in a guild. This means that if a user sends a message in one channel and then sends another message in a different channel, the bot will consider these messages as part of the same conversation. If you want to maintain separate conversation contexts for each channel, you would need to modify the conversation_id to include the channel ID as well as the guild ID.
